# myNews — Daily Morning Brief 📰

A bot that emails you a concise, up-to-date daily digest at **8:00 AM Pacific**:
AI innovation, new research & papers, politics, geopolitics, tech stocks, and a
daily "one thing to know." Content is generated by **Gemini 2.5 Flash with live
Google Search grounding** and delivered via **Gmail SMTP**. It runs on
**GitHub Actions**, so it works even when your computer is off.

## Setup (one time)

### 1. Get a Gemini API key
- Go to https://aistudio.google.com/apikey and create an API key.

### 2. Create a Gmail App Password
- Enable 2-Step Verification on your Google account.
- Go to https://myaccount.google.com/apppasswords, create an app password,
  and copy the 16-character value (no spaces).

### 3. Add GitHub repository secrets
In the repo: **Settings → Secrets and variables → Actions → New repository secret**.
Add all four:

| Secret name          | Value                                        |
|----------------------|----------------------------------------------|
| `GEMINI_API_KEY`     | your Gemini API key                          |
| `GMAIL_ADDRESS`      | `bobball281@gmail.com` (the sender)          |
| `GMAIL_APP_PASSWORD` | the 16-char Gmail app password               |
| `RECIPIENT`          | `abhinavballa28@gmail.com` (where it's sent) |

### 4. Test it
- Go to the **Actions** tab → **Daily News Digest** → **Run workflow**
  (keep "force" checked) to send yourself a test email immediately.

That's it. It will then arrive automatically every morning at 8am Pacific.

## Run locally (optional)

```bash
pip install -r requirements.txt
export GEMINI_API_KEY=... GMAIL_ADDRESS=... GMAIL_APP_PASSWORD=... RECIPIENT=...
python -m news_bot.main --force
```

## Customize

- **Sections / tone:** edit the `PROMPT` in `news_bot/generate.py`.
- **More depth:** set `GEMINI_MODEL=gemini-2.5-pro` (repo variable or env).
- **Delivery time:** change `SEND_HOUR_PACIFIC` in `news_bot/main.py` and the
  two `cron` lines in `.github/workflows/daily-news.yml` accordingly.

## How the 8am timing works

GitHub Actions cron runs in UTC and ignores daylight saving. The workflow
triggers at both 15:00 and 16:00 UTC; `main.py` checks the real
`America/Los_Angeles` time and only sends when it's 8am there — correct in both
PST and PDT. (GitHub may delay scheduled runs by 5–15 min under load.)
