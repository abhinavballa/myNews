# myNews — Personalized Morning Brief 📰

A bot that emails each user a concise, up-to-date daily digest **at their own
local delivery time**. Content is generated by **Gemini with live Google Search
grounding** and delivered via **Gmail SMTP**. It runs on **GitHub Actions**
hourly, so it works even when your computer is off.

Each user's brief is built from *their* `compiled_profile` (persona, edge focus,
and section list) stored in **Supabase** — not from a single hardcoded prompt.
**Phase 1** is the per-user worker (email delivery). **Phase 2** adds the
installable PWA: magic-link signup, a free-text interests box compiled into a
profile, and an in-app reader/archive. Web push is Phase 3. See
[`docs/superpowers/specs/2026-07-20-personalized-multiuser-digest-design.md`](docs/superpowers/specs/2026-07-20-personalized-multiuser-digest-design.md).

## How it works

- The worker (`news_bot/main.py`) runs **hourly**.
- Each run fetches all `active` profiles, keeps the ones whose current **local**
  hour equals their `delivery_hour` and who have **no digest yet** for their
  local date, then generates and delivers.
- A `UNIQUE (user_id, local_date)` constraint makes re-runs and delayed
  schedules idempotent — nobody is double-delivered.

## Setup (one time)

### 1. Create the Supabase project
- Create a project at https://supabase.com, then run
  [`supabase/schema.sql`](supabase/schema.sql) in the SQL editor.
- A trigger auto-creates a `profiles` row (blank `compiled_profile`) whenever
  anyone signs in, so no uuid is ever created by hand. For your own account,
  sign in once, then run [`supabase/seed.sql`](supabase/seed.sql) with your
  `auth.users` id to fill in your `compiled_profile` and `wants_email = true`.

### 2. Get a Gemini API key
- https://aistudio.google.com/apikey

### 3. Create a Gmail App Password
- Enable 2-Step Verification, then create an app password at
  https://myaccount.google.com/apppasswords (16 chars, no spaces).

### 4. Add GitHub repository secrets
**Settings → Secrets and variables → Actions → New repository secret**:

| Secret name                 | Value                                          |
|-----------------------------|------------------------------------------------|
| `GEMINI_API_KEY`            | your Gemini API key                            |
| `GMAIL_ADDRESS`             | `bobball281@gmail.com` (the sender)            |
| `GMAIL_APP_PASSWORD`        | the 16-char Gmail app password                 |
| `SUPABASE_URL`              | `https://<project>.supabase.co`                |
| `SUPABASE_SERVICE_ROLE_KEY` | project service-role key (bypasses RLS)        |

> The service-role key is powerful — keep it only in GitHub secrets, never in
> the PWA (Phase 2 uses the anon key + magic-link auth there).

### 5. Test it
- **Actions → Daily News Digest → Run workflow**, optionally passing a profile
  `user` uuid to force a single delivery immediately.

## Run locally (optional)

```bash
pip install -r requirements.txt
export GEMINI_API_KEY=... GMAIL_ADDRESS=... GMAIL_APP_PASSWORD=...
export SUPABASE_URL=... SUPABASE_SERVICE_ROLE_KEY=...
python -m news_bot.main --user <your-profile-uuid>   # force one user now
python -m news_bot.main                               # normal hourly behavior
```

## The PWA (Phase 2)

The installable web app lives in `public/` (static) plus one Python serverless
function in `api/`. It deploys to **Vercel** with zero build step.

**Layout**
- `public/` — the PWA (served at `/`): sign-in, onboarding, Today, Archive, Settings.
- `api/compile-profile.py` — `POST /api/compile-profile`: compiles the interests
  text box into a `compiled_profile` via Gemini and writes it back **as the
  signed-in user** (their token → RLS applies). This exists because a static page
  can't hold the Gemini key.

**Deploy**
1. Fill `public/config.js` with your `SUPABASE_URL` and **anon** key (safe to
   ship — RLS protects the data).
2. In Supabase → Authentication → URL Configuration, add your Vercel domain to
   the redirect allowlist (magic-link sign-in returns there).
3. Import the repo into Vercel and set function env vars: `GEMINI_API_KEY`,
   `SUPABASE_URL`, `SUPABASE_ANON_KEY`. Deploy.
4. Open the site → sign in → describe your interests → confirm the sections →
   pick a delivery hour. On iOS, **Add to Home Screen** to install (and, in
   Phase 3, to enable notifications).

The GitHub Actions worker and the PWA share the same `profiles`/`digests`
tables, so a profile created in the app is delivered by the hourly worker with
no extra wiring.

## Tests

```bash
pip install -r requirements-dev.txt
pytest
```

Covers due-user selection across timezones and DST boundaries, `compiled_profile`
schema validation, prompt compilation, interests-compilation parsing, and digest
idempotency.

## Customize

- **A user's sections / tone / persona:** edit that profile's `compiled_profile`
  in Supabase.
- **Delivery time / timezone / channels:** the profile's `delivery_hour`,
  `timezone`, `wants_email` columns.
- **More depth:** set `GEMINI_MODEL=gemini-3.5-pro`.
